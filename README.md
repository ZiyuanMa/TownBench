# TownBench

> 一个面向 LLM Agent 的微型实体经济与长期规划能力评测基准。
> (An LLM Agent benchmark simulating a small-town economy to test complex decision-making and long-term planning.)

---

## 1. 核心定位：什么样的一座“小镇”？

**TownBench 旨在测试大语言模型在受限生存环境下的“财商”与运筹能力。** 

现有的许多 Benchmark 通常在测试模型的单次问答、代码编写或是简单的网页检索能力。而 TownBench 提供了一个“开放式的生存与经营环境”：模型将附身于一个虚拟角色，通过纯文本交互，在一个陌生的小镇中摸爬滚打，依靠打工、交易、甚至情报整理来积累财富并活下去。

## 2. 小镇里有什么？

小镇并不是一个死板的任务清单，而是一个自洽的微型经济体，包含：
*   **空间与建筑 (Locations)**：小镇由不同的区域（如中心广场、商业街）和具体地点（如餐厅、商店、加工台）组成。Agent 必须在这些地点间“跑图”。
*   **可互动设施 (Objects)**：如“招聘公告板”（用于接取兼职）、“售货机”（用于购买补给）、“工作台”（用于消耗材料生产商品）。
*   **资源与物品 (Items & Resources)**：小镇里流通着各种实物，模型需要通过买卖、合成等方式处理它们。

## 3. Agent 的生存挑战

在这个世界里，模型没有“上帝视角”，且受到严格的物理和经济限制。模型必须时刻关注四大核心维度：
*   💰 **金钱（Money）**：用来购买门票、补给、坐车。这是游戏结束时最重要的评分指标之一。
*   ⚡️ **能量/体力（Energy）**：Agent 执行任何动作都会消耗能量。能量一旦归零，当前回合就会判定为“饿死”/失败。必须花钱买食物来恢复。
*   ⏳ **时间（Time）**：小镇中的每个动作都有时间成本，部分高薪任务甚至是限时的。
*   🎒 **背包（Inventory）**：Agent 无法无限囤积物品，背包容量有限，逼迫模型做好仓储管理。

**有限的视野 (Partial Observability)**：模型看不到全图全貌。它每一回合只能接收当前房间能看到的信息，必须像玩文字冒险游戏一样亲自去探索。

## 4. 模型怎么和小镇交互？

交互是一个最经典的文字调用闭环：
1. **看 (Observe)**：环境发送当前状态描述（例：“你在中心广场，前面有个公告板，左边有个商店”）。
2. **做 (Action)**：模型通过调用标准化的工具（Tool Calls）来行动，比如 `移动到(商店)`、`检查(公告板)`、`使用(购买面包)`。
3. **变 (Transition)**：环境判定动作是否成功（钱够不够？前提条件对不对？），然后扣减体力和时间，返回动作结果及新的环境文本。

## 5. TownBench 到底在考模型什么能力？

不同于传统的“测试集”，TownBench 被设计用来淘汰那些只能做短期判断的模型。它着重考验以下高阶能力：

*   **ROI 计算与权衡（算经济账）**：面对多个选项时，模型能否算出性价比？例如：“我是花 10 块钱坐快车去接个大单？还是走过去接个小单？”
*   **多步规划与防死锁（抗并发症）**：从“买生肉 -> 找炉子 -> 消耗体力做熟 -> 去集市卖掉”是一个长达十几步的生产链。模型能不能预留足够的金钱和体力，无报错地走完这个闭环，而不是半路“没钱开炉子”导致死锁？
*   **有效探索与抗干扰**：小镇里有高利润情报，也有纯粹的背景噪音（Distractors）。模型需要自己记笔记并筛选出有效信息。

## 6. 场景驱动的无限可能

TownBench 的底层引擎与具体的剧本完全解耦。整个小镇是通过简单的 YAML 配置文件（Scenario）驱动的。
这意味着，研究者只需要修改 YAML 配置，就能轻易扩展出“线性跑腿村”、“复杂加工厂”、“引入动态物价的商业街”等各种难度的 Benchmark。

## 7. 快速开始

> 详细的系统架构说明，请参考 `docs/architecture.md`。

安装依赖并配置环境：
```bash
python -m venv .venv
.venv/bin/pip install -r requirements.txt
```

运行测试用例以验证环境：
```bash
.venv/bin/python -m pytest
```

体验 OpenAI Agent 基准模型在默认场景中的表现：
```bash
# 请确保环境变量（如 .env）中已配置相应的 API Keys
.venv/bin/python scripts/run_openai_baseline.py --scenario scenarios/demo_town/scenario.yaml
```
